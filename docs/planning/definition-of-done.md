# Definition of Done (DoD)  
**Project:** [Project Name]  
**Version:** 1.0  
**Last Updated:** [Date]

A task/feature is considered DONE when ALL of the following are true:

- [ ] Code written and formatted per relevant skills (e.g., PSR-12, ES2025+)
- [ ] Follows Conventional Commits for git messages
- [ ] Artifacts saved correctly (/sql/, /examples/json/)
- [ ] No obvious bugs or security issues (agent self-check)
- [ ] Lint/validation gates pass (if applicable)
- [ ] Plan followed and approved by user (yes/no loop respected)
- [ ] /docs/context/current-status.md updated with summary
- [ ] Commit pushed (or ready to push)

Exceptions / waivers must be explicitly approved.
